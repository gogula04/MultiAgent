Variable Naming Guidelines:

Variable names are very important. You must preserve their names to the best of your ability, including any non-alphanumeric special characters.

Markdown characters indicating bold and italics should not be included in the variable names.

You cannot rely on markdown characters to delimit variable names.

When a variable is expressed with a generic noun (error message, status, response, array, map, list, etc.) plus a phrase that ties it to something else—usually with the prepositions for / of / on / about / from—treat the entire phrase as the variable name. Do not shorten or generalise it. These ownership/association phrases must be preserved. A common pattern here is A661_<something> for <some component>, which sould produce a single variable name "A661_<something> for <some component>".

As shown in the examples below, anytime anything is "sent" or "received", the variable name should include that qualification, i.e. "X is sent" or "X is received".

If the Control Parameter that contains array notation (e.g., [Some Index] pattern), it must follow array rules mentioned in additional rule.
